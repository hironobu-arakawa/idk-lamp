# Overview

この docs は、decision-closure リポジトリで扱っている
**判断閉路構造（Decision Closure Structure）** を、
作者自身が忘れないため、また第三者が追体験できるように
分解して記録するものです。

本リポジトリの関心は一貫して次の一点にあります。

> この判断は、AIが閉じてよいか。

正解・精度・倫理・UIは扱いません。
判断を「閉じてしまってよいかどうか」だけを、
構造として扱います。

┌──────────────────────────────────────────────┐
│                用途未定義世界                │
│  (5軸の値は観測できない / 免責はAI内部に置けない) │
└──────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────┐
│            判断の5軸（人間の判断構造）         │
│  相手 / 責任 / 精度 / 状況 / 時間              │
│  → 値は使えないため「危険条件」だけ抽出        │
└──────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────┐
│        判断閉路構造（Decision Closure）        │
│  AIが判断を「閉じてよいか」を決める純粋構造     │
│                                                │
│  【5つの閉路ステップ】                         │
│   1. 責任主体は1人か                           │
│   2. 心理的取消に依存していないか              │
│   3. 物理的に直接取り消せるか                  │
│   4. 行動が連鎖しないか                        │
│   5. 社会的文脈を帯びていないか                │
│                                                │
│  → 1つでも破綻したら Human-Closed              │
│  → すべて通過したら AI-Closed（極めて限定）    │
└──────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────┐
│                途中反転（Transition）          │
│  対話途中で5軸の危険条件が顕在化し、閉路が破綻 │
│                                                │
│  【途中反転の5分類】                           │
│   - 社会文脈反転（相手軸）                     │
│   - 責任反転（責任軸）                         │
│   - 可逆性反転（精度軸→取消可能性）            │
│   - 文脈反転（状況軸）                         │
│   - 時間反転（時間軸）                         │
│                                                │
│  → 即 Human-Closed に遷移                      │
└──────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────┐
│                 idk-lamp（シグナル）           │
│  判断が「できない」ではなく                     │
│  「閉じてはいけない」ために点灯する            │
│                                                │
│  【点灯理由の三層モデル】                      │
│   - 直接原因：途中反転の種類                   │
│   - 構造原因：閉路ステップの破綻               │
│   - 源泉：5軸の危険条件                        │
└──────────────────────────────────────────────┘


この docs は以下の順で読むことを想定しています。

1. 前提となる世界観（用途未定義世界）
2. 判断閉路構造そのもの
3. 対話途中で起きる反転（Transition）
4. 判断の5軸との対応関係
5. idk-lamp の位置づけ
